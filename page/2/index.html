how fix 65536 error in Eclipse

最近项目遇见65536error在Eclipse

  1. 合并jar包用ant 下载之后可以配置环境变量也可以到ant/bin执行,新建build.xml 内容如下:

自动把当前目录下所有的jar打包成一个hello1.jar

  1. 然后到Android SDK 目录build-tools 下找一个比较高版本在当前目录打开cmd 输入dx --dex --output=classes2.dex yourjarname.jar在当前目录就可以看到合并后的jar被转换为.dex文件
  2. 把step2中的.dex文件放到Eclipse的src目录下jar包引用进去.
  3. 下载android.support.Multidex.jar包.(可以通过在Android Studio中引用然后再gradle目录找到this.jar)
  4. 当前Application集成MultiDexApplication用法到官网 有
  5. 到此就可以运行了.如果出现jumbo错误在project.properties中加入dex.force.jumbo=true

ubuntu 开启root并打开ssh远程访问

首先安装

sudo apt-get install openssh-server提示没权限的话就 用root 登陆 su root 如果没有root 就用装完系统的第一个账号登陆 修改root密码 sudo passwd -root然后修改 文件vim /etc/ssh/sshd_config有一行关键字是nopassword 改成yes就好,然后重启ssh
