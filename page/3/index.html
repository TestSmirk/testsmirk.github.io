自动把当前目录下所有的jar打包成一个hello1.jar

  1. 然后到Android SDK 目录build-tools 下找一个比较高版本在当前目录打开cmd 输入dx --dex --output=classes2.dex yourjarname.jar在当前目录就可以看到合并后的jar被转换为.dex文件
  2. 把step2中的.dex文件放到Eclipse的src目录下jar包引用进去.
  3. 下载android.support.Multidex.jar包.(可以通过在Android Studio中引用然后再gradle目录找到this.jar)
  4. 当前Application集成MultiDexApplication用法到官网 有
  5. 到此就可以运行了.如果出现jumbo错误在project.properties中加入dex.force.jumbo=true

tencent 互动直播接入

  • 需求
    让本身的应用中具有直播功能并且具有交互.
  • 过程
    由于对这方面一点经验都没有,盲目的下demo去看源码,代码写的并没有希望的那样通俗易懂,并且公司总是断网,严重影响心情.

  1. 之前Tencent的互动直播sdk是1.7V,然而昨天开始已经1.8(并没有在官网公开只有在github)文档从开始就没有特别清晰的思路.

  2. 其实没有第二点了…

今天深入看了一下安卓的源码,发现直播房间要在自己的服务端维护,Tencent给了一个php的服务端代码,今天把闲置的vps激活,然后装上httpd Apache,php mysql运行了一下效果蛮好,当我在移动端创建房间的时候有不能成功的返回房间id,因为第一次接触php代码,定位问题很慢,找了俩小时,发现不能写入数据到db未完待续.

移动端这边只要调用一些方法,比如enterRoom(),exitRoom(),然后初始化硬件就ok至于登陆的话完全是imsdk.imsdk在这个互动直播sdk中占了很大一部分.
这次集成第三方,真的是考研代码的阅读能力,感觉在这次集成中有了一些小小的进步,也距离自己全栈的路更进一步,
